## 已落地改动

### 1. 修复 `Today` 页面过期问题

- 首页与多语言首页已禁用静态预渲染，改为请求时生成当天数据。
- 城市 sunrise / prayer 详情页已从 `getStaticPaths` 切换为基于路由参数的请求时查找。
- 页面响应已补充缓存头，兼顾 freshness 与边缘缓存。

### 2. 重构相关推荐与内部链接

- 新增统一的地理距离计算与 nearby city 排序逻辑。
- 详情页已改为优先展示真实邻近城市，而不是跨洲随机城市。
- 对比城市列表已优先取同国家候选，提升页面主题相关性。

### 3. 增强页面独特价值

- 英文 sunrise 页已新增 30 天前后对比：日照时长、日出变化、日落变化。
- 英文 prayer 页已新增 30 天前后对比：Fajr / Maghrib 变化与 daylight 变化。
- 中文 sunrise / prayer 页、西语 prayer 页也已补上相同层级的 30 天变化模块。
- 法语与土耳其语的 sunrise / prayer 页也已补上相同层级的 30 天变化模块。
- 阿拉伯语 sunrise / prayer 页也已补上相同层级的 30 天变化模块。
- 英文、中文、西语、法语、土耳其语、阿拉伯语 sunrise / prayer 页现已补齐年度极值与季节节点表，内容从“今天 / 30 天”扩展到“全年”。
- 第一版完成定义已经满足；后续增强重点将转向年度极值、季节段变化等更强指标。

### 4. 降低详情页重复负载

- Footer 不再在每个详情页重复注入完整城市搜索 JSON。
- 搜索数据已拆到 `/city-search.json`，由前端按需拉取并复用。
- 这一步直接降低了详情页 HTML 冗余，改善大规模抓取时的负载成本。

### 5. 建设可链接内容资产

- `daylight-length` 主题 guide 已新增真实城市对比数据表，而不是纯导航文案。
- `prayer-times` 主题 guide 也已新增真实城市对比数据表和 30 天变化摘要。
- `golden-hour` 与 `moon-phase` 主题 guide 现已补齐城市对比表、快速结论摘要与方法/API 复用入口。
- guide 页面已新增 `Fast takeaways` 摘要层，降低引用和转述成本。
- guide 页面已新增 `Method and reuse` 入口，把方法页、API 文档和示例响应串起来。
- 第一版完成定义已经满足；后续会继续扩更多 guide 资产。

### 6. 建立外链增长基础设施

- 已新增 `GET /api/v1/times?city=<slug>&date=YYYY-MM-DD` 接口。
- 已新增 `GET /api/v1/cities?query=<term>&limit=<n>` 城市发现接口，降低外部接入门槛。
- 已新增 `GET /api/v1/openapi.json` 机器可读描述，方便后续生成客户端或接入内部工具。
- 已新增 `/guides/api/` 文档页，说明请求方式、返回字段和入口。

## 完成定义

### 1. 修复 `Today` 页面过期问题

- 首页和多语言首页不再输出构建时冻结的“今天”数据
- `/sunrise/[city]/` 和 `/prayer-times/[city]/` 及对应语言版本不再输出过期日期
- 线上页面访问时返回的日期与时间与当天一致

### 2. 重构相关推荐与内部链接

- 不再使用跨洲随机城市推荐
- 详情页的相关推荐至少优先命中：最近城市、同国家城市、同区域城市
- 推荐结果稳定且可复现

### 3. 增强页面独特价值

- 至少新增一组强数据型内容块，而不是泛文案扩写
- 内容可验证、按城市显著不同、能帮助用户比较或决策

### 4. 降低详情页重复负载

- 详情页 HTML 不再重复内联完整城市搜索 JSON
- 搜索数据改为独立资源，并可被前端复用

### 5. 建设可链接内容资产

- 至少落地一个更像“资产”而不是“导航桥”的 guide
- 页面应包含可引用的数据、表格、方法说明或下载/接口入口

### 6. 建立外链增长基础设施

- 站内存在可对外介绍的 API / 文档入口
- sitemap / 站内导航能把资产页与开发者入口串起来
